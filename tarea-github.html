Actividad Práctica

Control de
Versiones con
GitHub

Aprende a crear y administrar repositorios digitales, comprender cómo se almacenan archivos, registrar cambios y documentar el avance de un proyecto mediante commits en la plataforma GitHub.

01 Propósito
🎯

Objetivo Principal

Familiarizarse con el uso de herramientas de control de versiones mediante GitHub, ampliamente utilizada en el desarrollo de software para almacenar proyectos, organizar archivos y registrar cambios a lo largo del tiempo.

💡

¿Qué aprenderás?

  • Crear y administrar un repositorio digital
  • Entender cómo se almacenan los archivos
  • Registrar cambios mediante commits
  • Documentar el avance de un proyecto
  • Comprender el valor del versionamiento
02 Pasos a seguir
1

Crear cuenta en GitHub

Si aún no tienes una cuenta, regístrate en la plataforma GitHub. Este será tu espacio de trabajo para toda la actividad.

2

Crear el repositorio público

Crea un repositorio con el nombre Proyecto-Versionamiento-NombreApellido, sustituyendo tu nombre y apellido. El repositorio debe ser de acceso público.

3

Configurar el archivo README.md

Al crear el repositorio, agrega un archivo README.md. En él debes escribir:

  • › Tu nombre completo
  • › La sección a la que perteneces
  • › El nombre del curso
  • › Una breve descripción del proyecto
4

Crear el proyecto digital

Desarrolla una página web básica en HTML que incluya:

  • › Información personal general
  • › Al menos una imagen
  • › Una lista de intereses o gustos personales
  • › Cualquier contenido adicional que desees agregar
5

Subir archivos y primer commit

Sube los archivos del proyecto al repositorio y registra el primer cambio con un commit que tenga un mensaje claro describiendo la acción realizada.

a1b2c3d Subida inicial del proyecto HTML
6

Realizar al menos 4 actualizaciones adicionales

En distintos momentos, realiza 4 o más cambios visibles al proyecto. Cada cambio debe registrarse con un commit descriptivo. Ejemplos de cambios:

Agregar nueva sección
Modificar contenido existente
Incorporar imágenes adicionales
Mejorar la organización
Agregar nuevas páginas HTML
03 Actualización del README.md
📄

Documenta la evolución del proyecto

A lo largo del desarrollo, el archivo README.md debe actualizarse continuamente para reflejar:

  • Una descripción más detallada del propósito del proyecto
  • Los cambios realizados durante cada actualización
  • Las fechas en que dichos cambios fueron efectuados

Este archivo funciona como un documento de referencia que permite comprender la evolución del proyecto con el paso del tiempo.

06 Conclusión
🚀

¿Por qué importa el control de versiones?

Esta actividad tiene como finalidad que comprendas la importancia del control de versiones en el desarrollo de proyectos digitales. El uso adecuado de herramientas como GitHub permite:

  • Organizar la información de manera eficiente
  • Evitar la pérdida de archivos importantes
  • Facilitar el seguimiento de los cambios realizados
  • Documentar el avance de un proyecto a lo largo del tiempo
